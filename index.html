DIS Printing Manual

There are generally 3 different methods for connecting printers to the DIS business system.
*LAN - This method uses the HP Jet Direct protocol also known as Raw printing and is sent to TCP port 9100 of the printer. It has communication capabilities that will inform the business system of errors, out of paper conditions, jams etc..
LPR - Line Printer Remote, uses the LPD service (port 515) to communicate to the device. This method does not provide any communication to the system about it's status. This method is used where devices do not support the *LAN configuration. Please note that LPD is a legacy technology and not a formal standard. Vendors have implemented slight variations in how devices use the LPD service. This can cause compatability issues from device to device.
KPS - The Keystone client is configured to connect a supported USB host connected printer to the business system. This is generally only supported with Lexmark and HP devices.

IBM publishes a document on supported printers and suggested methods for communication. The list of printers is several years behind what is currently in production, but it can be used as a guide to successfully set up a printing device.

For consistency use a Terminal session such as a "Green Screen" or a Quantum session as the interface is almost identical.
This is the WRKDEVD *PRT view of the printing devices installed on the system.
You can open this view by the key combo ALT+A entering the command WRKDEVD *PRT and clicking OK, or opening from the Quantum home screen System -> System Operator Only -> Start/End Printers

When using Quantum to access the IBM terminal session, type the command WRKDEVD *PRT on the ==> command line to Work with Device Descriptions.


Installing a printer device

There are generally three methods for installing a printer.
*LAN - Networked device using HP Jet Direct protocol port 9100. Provides bi-directional communication to the business system for device errors.
LPR - Networked device using LPD (LPR) protcol port 515.
KPS - USB connected printer using a Keystone session to connect to business system.

The best practice is to copy an existing printer that matches as closely as possible the new printer make and model. **If there are no *LAN devices to copy please contact DIS support.**
Select a printer from the list using page up and down and place a 3 in the options field to copy it.



In the Device description field enter the new DIS ID and hit enter. The screen will change and show additional settings. Use page down twice to see the next screen to modify the IP address.



Enter the new IP address and pay close attention to the single quotes. Page down again



Enter the printer description including make and model and pay close attention to the single quotes. Hit the enter key when finished.


The message that the device was created successfully.



Use F5 or Fn+F5 to refresh the list of printer devices and E2 is added to the list.



Place 8 in the options field next to E2 to get to the screen and VARY ON the device.



Depending on your user, you may have to increase the "Assistance Level" by using Shift+F9(F21) or Fn+Shift+F9(F21)

Scroll down to the bottom of the file to the "PrinterDevice="" section. Append the name of the DIS ID to that line, then save and close. For example PrinterDevice=E5. If the configuration is to be removed, restore the line to PrinterDevice=



Open the second file legasuite.ini and scroll to the bottom. Type DEFAULT on the line next to "Default Printer=". Save and close the file. For example, DefaultPrinter=DEFAULT. In the event that the configuration os to be removed, change the line back to DefaultPrinter=

Verify that the device name and the printer selected from the list is correct and click Connect. The window will minimize back to the system tray and the printer should now be ACTIVE and VARIED ON.



Removing a *LAN printer device

Writers must be stopped and *LAN devices will also need the device VARIED OFF before it can be completely removed. The output queue will also have to be empty for any device, *LAN, LPR and KPS before removal. Type WRKOUTQ at a command line and page down to the printer you want to remove.

Type ENDWTR on the command line to start a dialog where you can enter the DIS ID of the writer you want to stop. This screen is helpful if you have more than one writer you wish to stop.



Enter the DIS ID of the writer and hit the ENTER key.



The command to stop the writer can also be run all at once. For example ENDWTR E3.



Now that the writer has been stopped. Place a 5 next to the device that you want to manage the spools and remove.



Place a 4 on the line next to all of the documents in the queue and hit the Enter key.



Hit the Enter key a second time to confirm deletion.



There should be no spools in the queue. If there are, repeat the steps to remove them.



Return to the Work with Device Descriptions screen again and page down to the desired device and place a 4 in the Opt column next to it and hit the Enter key.



Hit the Enter key a second time to confirm deletion making sure it is the correct device ID.



Use F5 to refresh if necessary and now the E4 printer is no longer in the list.



Removing a LPR printer device

Writers must be stopped before it can be completely removed.
Type the command ENDWTR (DIS ID), for example ENDWTR E3. Type the command WRKOUTQ. Verify that the writer is stopped and that the outq is empty.



Return to the Work with Device Descriptions screen and page down to the device you want to remove. Place a 4 in the Opt column next to the device you wan to remove.



Hit the Enter key to confirm the removal of the device.



Device E3 has been removed from the list and a message at the bottom of the screen confirming that.



Removing a KPS printer device

Troubleshooting

Sudden Print Stoppage

When printing stops suddenly it is normally one of three things.
1) A MSGW has occured due to a form change.
2) The IP address of the device has somehow changed either by a power spike/sag or firewall reboot if the printer has a dynamically assigned IP address. Modify IP
3) A printers Writer process has stopped/closed. Check the WRKOUTQ and start the Writer

*LAN Troubleshooting

Removal of a *LAN printer

A common error when removing a *LAN device. "Device not deleted. Device in use." This is caused by the device still VARIED ON or the WRITER is still running. Use the ENDWTR command to stop and then enter an 8 first and then a 2 to VARY OFF. Attempt the operation again.

KPS/Keystone Troubleshooting

KPS printing has several moving parts that all have to be configured and operating properly to work. If there is a print stoppage try verifying the following.
1) Type the command WRKOUTQ at a command line and verify that the device has the writer running in the Writer column. If it does not, that would indicate that the Keystone session supporting the connection on the host PC is not running.
2) The Universal Print Driver is no longer selected in the printers advanced tab. Open Windows Printer & Scanners and select the printer and go to properties to verify that the driver is selected in the list.
3) The TN5250 emaulator is disconnected. Right click on the printer icon in the system tray and restore to the Desktop. Verify the button at the bottom says disconnect, if not click on connect and the screen will switch to gray indicating that it is enabled.

Printer Messages (MSGW)

Messages are generated due to form changes, printing an invoice and following up with a check for example. They are also generated when a *LAN device is first created and sent it's very first print job. The can also happen when a printer is no longer available at the network address that is configured on the business system.

Check printing is designed to produce a MSGW due to the likleyhood that check stock would have to loaded into the printer before printing. A MSGW pauses the job to provide an opportunity to do this. **NOTE that a LPR check printer will NOT produce a MSGW and the stock will have to loaded in advance.

This message is from a form change. Normally the correct option is "G" which will continue to print the current form until it changes again at which time "G" should be used again and so on.
The only exception would be for printing STATEMENTS. In this case to correct option would be "I". After the STATEMENTS finish printing the system will continue to print out the previous form type avoiding another MSGW to answer.



This MSGW is indicating that the printer is not in communication. This can be caused by the printer's IP address changing or stops responding and requires power cycling the device to resolve.



Report Center

Quantum Report Center provides access to the DIS printing output queues. It can be helpful to view printed documents electronically using a PDF viewer and can serve as a work-around if the printer is temporarily out of communication with the business system such as an IP address change for example.

From the home screen, click on System and then Quantum Report Center



A new tab will open taking you to the interface by default showing only the jobs of the user in the session and all output queues. Take note of the row of tabs at the top. These are used to filter by user, output queue form type etc..



By changing the user from *CUR to *ALL and leaving the rest of the filters in place and clicking Apply, the interface will display all of the print spools on the system. Depending on the number of print spools on the system it can take several seconds to minutes to render. You may want to avoid using this filter if the interface hangs or becomes unresponsive. It is a good idea to make sure that none of the outpout queues contain an excessive number of files. Virtual printer queues such as ZZ, NP or NO PRINT should have no more that 2,000-3,000 spools. Type the command WRKOUTQ at a prompt to see the numbers.


Printer Configuration

The instructions are designed to be a guide to printer configuration and some screens can be slightly different than what is depicted. The concepts are the same none the less.

Lexmark

The easiest method for making configuration changes are through the web interface of a network connected printer. Lexmark by default will display the IP address of the printer on your local network.
Put this IP address into the address bar of a web Browser. You can encounter a warning page when the page is first loading indicating the website is a risk. This isn't the case, any device being accessed
by IP address will generate this error.** BE AWARE THAT THE PRINTER IP CAN LOOK DIFFERENT THAN THE IP IN THE DIS NETWORK CONFIGURATION **. This is due to a 1:1 NAT or network address translation. What we are concerned with is the last set of numbers or octet, for example xxx.xxx.xxx.100. It is this last set that will be the same on the business system and on your local network.

A Lexmark printer web interface at the home screen. Click on the Network/Ports menu link on the left margin.



Scroll the page down a little and look for the Ethernet menu item in the center of the screen.



Ethernet will expand showing a link in blue text to "Set Static IP Address", click on that.



This is where you can set the printers IP address statically so that it can be retained. Look at the printers description in WRKDEVD *PRT to get the last octet of the IP. Click Save to set the new static IP. You should now be able to access the printers web interface at the new IP as well. This serves as a good test that the process was done correctly.



HP

HP printers do not by default display their IP address. To get this information it will be necessary to print a configuration report.
At the printers panel, press the settings button, it may look like a gear or a wrench icon. When in this menu use the arrow buttons to advance through until you see the "Reports" menu and press the "OK" button. Use the arrow buttons again to advance to the "Network Configuration" or "System Summary" report. Click OK to select and print.
Look for the IPv4 IP address. It should start with "10." or "192.168" in most cases. If you see an IP address starting with "169", this is a self assigned IP and happens when the printer can not get a dynamically assigned IP from the firewall. This is normally caused by a conectivity issue such as a faulty cable.
